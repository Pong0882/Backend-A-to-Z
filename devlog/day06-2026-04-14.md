# Day 06 — 2026-04-14

## 오늘 할 것

### 1. Redis 도입 — Refresh Token 전환

**목표:** MySQL에 저장 중인 Refresh Token을 Redis로 이전

**순서:**
1. `docker-compose.yml`에 Redis 컨테이너 추가
2. `build.gradle`에 `spring-boot-starter-data-redis` 의존성 추가
3. `application.yml` Redis 연결 설정
4. `RefreshTokenRepository` — JPA → Redis(`StringRedisTemplate` 또는 `RedisRepository`) 전환
5. 기존 `refresh_tokens` 테이블 대체 확인

**기대 효과:**
- TTL 자동 만료 — 지금은 로그아웃 시 직접 삭제해야 함. Redis TTL 설정하면 만료 토큰 자동 정리
- DB 부하 감소 — 토큰 조회/저장이 Redis 메모리로

### 2. 성능 비교 — DB vs Redis 응답시간 측정

**목표:** Redis 도입 효과를 숫자로 확인

**방법:**
- 전환 전 DB 조회 응답시간 측정
- 전환 후 Redis 조회 응답시간 측정
- 동일 요청 반복 (JMeter 또는 k6)

**측정 지표:**
```
평균 응답시간 (ms)
P95 응답시간
초당 처리량 (TPS)
```

### 3. @Cacheable 현재가 캐싱

**목표:** 한투 API 레이트 리밋 방어 + 반복 요청 성능 개선

**순서:**
1. `@EnableCaching` 활성화
2. `getStockPrice()`에 `@Cacheable` 적용 (TTL 5~10초)
3. `getDailyPrices()`에 `@Cacheable` 적용 (TTL 길게 — 일봉은 자주 안 바뀜)
4. 캐시 히트/미스 로그 확인

**캐시 저장소:** Redis (이미 띄운 것 재사용)

---

### 4. HikariCP 커넥션풀 모니터링

**목표:** DB 커넥션풀 상태를 Actuator로 확인

**순서:**
1. `application.yml` HikariCP 설정 추가
2. `/actuator/metrics/hikaricp.connections.active` 확인
3. 부하 테스트 시 커넥션 사용률 변화 관찰

```yaml
spring:
  datasource:
    hikari:
      maximum-pool-size: 10
      minimum-idle: 5
      connection-timeout: 30000
```

### 5. @Async 비동기 처리 — 일봉 수집

**목표:** `fetchAndSaveDailyPrices()` 호출 시 즉시 응답 반환, 백그라운드에서 수집

**순서:**
1. `@EnableAsync` 활성화
2. `ThreadPoolTaskExecutor` 빈 설정
3. `fetchAndSaveDailyPrices()`에 `@Async` 적용
4. 즉시 응답 반환 확인 (jobId 또는 "수집 시작됨" 메시지)

**기대 효과:**
- 지금: 요청 → 수초 대기 → 응답
- 이후: 요청 → 즉시 응답 → 백그라운드 수집
