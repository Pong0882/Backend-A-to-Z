# Day 07 (인프라) — 2026-04-15

## 오늘 한 것

- 컴퓨터 재시작 후 Multipass IP 변경으로 모니터링 서버 접속 불가 문제 발생
- Multipass VM 고정 IP 설정 (Hyper-V 내부 스위치 + netplan)
- pong-server: `192.168.100.10` 고정
- monitoring-server: `192.168.100.20` 고정

---

## 문제 상황

컴퓨터를 껐다 켜면 Multipass VM의 IP가 바뀐다.
`monitoring/prometheus.yml`의 pong-server 타겟 IP가 예전 IP로 되어 있어서 메트릭 수집이 끊겼다.

```
# .env 의 PONG_SERVER_IP가 이전 IP로 되어 있었음
PONG_SERVER_IP=172.24.126.133  → 실제 IP는 172.29.141.222 로 바뀌어 있었음
```

**임시 해결:** `.env`에서 IP 수동 변경 후 `bash start.sh` 재실행

**근본 해결:** VM 고정 IP 설정

---

## 해결 방법 탐색 과정

### 검토한 방법들

**1. pong-server.local (mDNS)**
- Windows에서 `ping pong-server.local` 시도 → `218.38.137.27` 외부 IP로 잘못 해석
- VM 안에서 `ping pong-server.local` 시도 → `Temporary failure in name resolution`
- Windows + Multipass 환경에서 mDNS가 동작하지 않음 → **불가**

**2. VM 이름으로 직접 ping**
- `ping pong-server` → `호스트를 찾을 수 없습니다` → **불가**

**3. /etc/hosts 직접 등록**
- IP가 바뀌면 결국 수동 수정 필요 → 근본 해결 아님 → **기각**

**4. pong-server가 자기 IP를 monitoring-server에 알려주는 방식**
- pong-server 시작 시 monitoring-server SSH 접속 후 `.env` 업데이트
- 문제: monitoring-server IP도 바뀌어서 접속 대상을 모름 → **동일 문제 반복**

**5. Hyper-V 내부 스위치 + 고정 IP**
- 별도 내부 네트워크를 만들어서 VM들에 고정 IP 부여
- 재시작해도 이 네트워크 IP는 바뀌지 않음 → **채택**

---

## Hyper-V 내부 스위치 + netplan 고정 IP 설정

### 전체 구조

```
Windows (호스트)
├── Hyper-V 가상 스위치: multipass-internal (내부 타입)
│   └── 호스트 측 IP: 192.168.100.1 (게이트웨이 역할)
│
├── pong-server VM
│   ├── eth0 — 기존 Multipass 기본 스위치 (재시작마다 IP 바뀜, 인터넷 연결용)
│   └── eth1 — multipass-internal 스위치 (고정 IP: 192.168.100.10) ← 신규
│
└── monitoring-server VM
    ├── eth0 — 기존 Multipass 기본 스위치 (재시작마다 IP 바뀜)
    └── eth1 — multipass-internal 스위치 (고정 IP: 192.168.100.20) ← 신규
```

- `eth0`: 인터넷 연결용. Multipass가 자동으로 관리. 재시작마다 IP 바뀜
- `eth1`: VM 간 내부 통신 전용. 우리가 직접 고정 IP 부여. 재시작해도 고정

### Step 1 — Hyper-V 가상 스위치 생성

Hyper-V 관리자 → 가상 스위치 관리자 → 새 가상 네트워크 스위치 → **내부** 선택 → 이름: `multipass-internal`

### Step 2 — 호스트 측 IP 설정

네트워크 연결 → `vEthernet (multipass-internal)` → 속성 → IPv4
- IP 주소: `192.168.100.1`
- 서브넷 마스크: `255.255.255.0`
- 기본 게이트웨이: 없음

### Step 3 — Multipass VM에 스위치 연결

PowerShell (관리자):

```powershell
# multipass-internal을 브리지 네트워크로 지정
multipass set local.bridged-network=multipass-internal

# 각 VM에 브리지 네트워크 연결
multipass stop pong-server
multipass set local.pong-server.bridged=true
multipass start pong-server

multipass stop monitoring-server
multipass set local.monitoring-server.bridged=true
multipass start monitoring-server
```

VM에 `eth1` 어댑터가 새로 생긴다.

### Step 4 — VM 안에서 netplan으로 고정 IP 설정

**pong-server:**

```bash
sudo nano /etc/netplan/60-eth1-static.yaml
```

```yaml
network:
  version: 2
  ethernets:
    eth1:
      addresses:
        - 192.168.100.10/24
```

```bash
sudo chmod 600 /etc/netplan/60-eth1-static.yaml
sudo netplan apply
ip addr show eth1
# inet 192.168.100.10/24 확인
```

**monitoring-server:**

```bash
sudo nano /etc/netplan/60-eth1-static.yaml
```

```yaml
network:
  version: 2
  ethernets:
    eth1:
      addresses:
        - 192.168.100.20/24
```

```bash
sudo chmod 600 /etc/netplan/60-eth1-static.yaml
sudo netplan apply
ip addr show eth1
# inet 192.168.100.20/24 확인
```

### Step 5 — .env 업데이트

monitoring-server에서:

```bash
cd ~/Backend-A-to-Z/pong-to-rich/monitoring
sed -i 's/PONG_SERVER_IP=.*/PONG_SERVER_IP=192.168.100.10/' .env
bash start.sh
```

## 트러블슈팅

### multipass set local.pong-server.networks 명령어 실패

```
set failed: Unrecognized settings key: 'local.pong-server.networks'
```

**원인:** Multipass 1.16.1에서는 `networks` 키가 아니라 `bridged` 방식으로 처리함

**해결:** `local.bridged-network` 먼저 지정 후 `local.VM이름.bridged=true` 사용

### netplan 파일 권한 경고

```
Permissions for /etc/netplan/60-eth1-static.yaml are too open.
```

**원인:** `nano`로 생성 시 기본 권한이 644 (others 읽기 허용)인데 netplan은 600 이하를 요구

**해결:** `sudo chmod 600 /etc/netplan/60-eth1-static.yaml`

### grafana.pongtrader.pro 502 에러

**상황:** 고정 IP 설정 완료 후 `grafana.pongtrader.pro` 접속 시 502

**원인:** pong-server의 `/etc/cloudflared/config.yml`에 monitoring-server IP가 예전 IP로 하드코딩되어 있었음

```yaml
- hostname: grafana.pongtrader.pro
  service: http://172.24.122.19:3000  # 예전 IP → 502
```

**해결:** 고정 IP로 변경 후 cloudflared 재시작

```yaml
- hostname: grafana.pongtrader.pro
  service: http://192.168.100.20:3000  # 고정 IP → 정상
```

```bash
sudo systemctl restart cloudflared
```

**배운 점:** cloudflared config에도 IP가 박혀있었다. VM IP가 바뀌는 환경에서는 IP를 직접 쓰는 곳이 어디든 전부 고정 IP로 교체해야 한다.

## 결과

```
pong-server    eth1: 192.168.100.10 (고정)
monitoring-server  eth1: 192.168.100.20 (고정)
```

이제 컴퓨터를 재시작해도:
- `eth0` IP는 여전히 바뀜 (인터넷 연결용, 무관)
- `eth1` IP는 고정 → prometheus.yml 타겟이 영구적으로 유효

`.env`의 `PONG_SERVER_IP=192.168.100.10`은 다시 바꿀 필요 없음

---

## netplan `version: 2` 가 뭔가

netplan 설정 파일 스키마의 버전. 네트워크 프로토콜 버전이 아니다.
현재 Ubuntu에서 사용하는 netplan 설정 포맷이 version 2이고, version 1은 구버전이라 현재 거의 안 쓴다.
